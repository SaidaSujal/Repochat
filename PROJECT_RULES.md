# RepoChat Project Rules

## Mandatory Rules
1. Never hardcode secrets.
2. Use .env for all credentials.
3. Add .env to .gitignore.
4. Use environment variables only.
5. Public GitHub repositories only.
6. No code execution from repository content.
7. Every answer must include source citations.
8. Validate all user inputs.
9. Add rate limiting to APIs.
10. Follow security-first development.

## Development Rules
- Modular architecture.
- Type hints.
- Clear folder structure.
- Reusable components.
- Meaningful logging.
